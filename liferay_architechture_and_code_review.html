Why Us ?

Custom Portlets

Liferay when used as a platform to develop business application, will have several custom portlets. The architecural mechanisms used to construct these portlets will decide how resilient your application will be to changes.

We study the custom portlets implemented in your application and review the structure of the same. Custom portlets form the bulk of your application if you have selected Liferay as a platform to implement your portal. The design must be such that it conforms to Open-Closed-Principle so that future enhancements and maintenance works like a charm.

Architectural components that we review are :

  • Responsiveness In User Interface
  • Browser Compatibility of custom JavaScript code used
  • MVC frameworks integrated like Spring MVC, Struts, JSF etc.
  • Mechanism used to integrate with Business Tier
  • Design of services implemented using Liferay Service Builder
  • Transaction management techniques used
  • Liferay Security & Permissions framework used & customizations done
  • Concurrency features implemented
  • Remoting (Web Services / Restful Web Services)
  • Workflow & ESB implemented
  • Auditing, logging & error handling mechanisms used

Liferay Hooks & Ext

Anything can be customized in Ext does not mean everything must be customized in Ext.

It is immensely important that for each customization task on hand, you ask - "Hook or Ext". The wrong choice can play a spoilsport in maintenance and migration to new version of Liferay.

This is what we review -

  • What is implemented in EXT was actually supposed to be in EXT and couldn't have been done through Hooks
  • The components used in EXT and its impact on migrating to newer versions of Liferay
  • The detailed level design of the code and its structure

Performance Review & Detailed Code Review

Data Structures used :

  • Most of the performance problems we have experienced in applications are due to inappropriate use of data structures by programmers. We have come across innumerable number of programmers with 3 to 4 years of experience not knowing the Collection type data structures beyond ArrayList and HashSet

Concurrency Code :

  • Flaws in applications with multi threading code is hard to detect. We do a through code review to ensure that Multi Threading is implemented as per the standards

Caching Mechanism :

  • Overuse and underuse of Caching mechanism can both hurt performance and quality. We look for places where cache could have been effectively used and those places where caching in-turn would turnout to be a culprit.

Objects Churned in Memory :

  • We identify where Singletons could have been used, where lazy initialization would have helped, where instance pooling would have added to the benefit.

Usage of Out-Of-the-Box Components of Liferay :

  • Liferay has innumerable components, utility and helper classes that are proven and can speed up the development work. We review and ensure that these are appropriately used.

Design Principles & Patterns :

  • Adherence to OO principles like OCP, ISP, DIP, SRP, Acyclic Dependency, Common Reuse, Common Release. Also we review where design patterns could have been adopted for more reusability and ease of maintenance.

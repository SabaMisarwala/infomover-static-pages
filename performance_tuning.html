Solution Strategy

Process

Given a project for performance tuning, we start with listening to you first.

Understanding the problem that is currently being experienced is a great place to start with. We analyze the as-is architecture of your application and bottlenecks that are hampering the performance. We move on to study the test results or preparing the required tests scripts in order to obtain the test results before tuning. Once there is concurrence on performance targets, we proceed with the following

Tasks

  • We start with analyzing the GC Logs and Heap Dumps.
  • We probe for possible Memory leaks, reasons of inefficient GCs and deadlocked & starving threads.
  • If the cause is identified we validate the same before suggesting a solution.
  • In case of memory leaks, we identify the exact leaking objects so that the development team can fix it
  • We do architecture and code review and suggest the areas of improvements where applicable for your development team to implement.
  • We do the code review to check the usage of data structures, algorithms and concurrency related code. These aspects, if not properly coded can have adverse effect on the performance of your application.
  • We finally suggest the best possible tuning parameters for your JVM so that it performs at its optimal level (There are more flags with JVM than at United Nations - lets us figure out which ones would boost the performance of your application)
